# 定价模型

> **内部文档，不可对外公开。** Token 与积分汇率（CPT）为核心商业机密。

## Kimi-K2.6 API 成本（SiliconFlow）

| 类型 | 价格 |
|------|------|
| 缓存命中 | ¥0.001100/K tokens (¥1.10/M) |
| 输入 | ¥0.006500/K tokens (¥6.50/M) |
| 输出 | ¥0.027000/K tokens (¥27.00/M) |

输出是输入的 **4.15 倍**。

## 积分计费公式

```
积分消耗 = max(1, int(estimated_tokens / 1000 × CPT))
```

- `estimated_tokens = 字符数 × 1.5`（中文字符约 1.5 token）
- **CPT（CREDITS_PER_1K_TOKENS）= 每 1000 token 消耗多少积分**
- CPT 存储在 `system_config` 表，管理后台可动态调整，重启后自动从 DB 加载

## 当前 CPT

**CPT = 20**（数据库 `system_config` 表，key = `credits_per_1k_tokens`）

## 利润率公式

```
利润率 = 1 - (API成本/积分 × 积分消耗) / 套餐价格
       = 1 - (0.0288 / CPT) / (价格 / 总积分)
```

保本线：`CPT ≥ 0.0288 × 总积分 / 价格`

各套餐保本 CPT：

| 套餐 | 保本 CPT |
|------|---------|
| 体验包 | 3.49 |
| 基础包 | 3.01 |
| 标准包 | 3.41 |
| 专业包 | 3.88 |
| 论文季卡 | 4.05 |

**安全下限：CPT 不得低于 4.05**（论文季卡保本线）。当前 CPT=20，利润空间充裕。

## 调价指南

CPT 是唯一的"利润旋钮"，套餐价格和积分固定不变：

- **CPT ↑** → 每篇消耗更多积分 → 用户套餐"不经用" → 利润率 ↑
- **CPT ↓** → 每篇消耗更少积分 → 用户套餐更划算 → 利润率 ↓
- **安全范围：CPT > 4.05**

| CPT | 体验包利润率 | 论文季卡利润率 |
|-----|------------|--------------|
| 5 | 33% | 33% |
| 10 | 67% | 67% |
| 15 | 78% | 78% |
| **20（当前）** | **83%** | **80%** |
| 30 | 89% | 87% |
| 50 | 93% | 92% |

## 新人赠送

`NEW_USER_BONUS_CREDITS = 200`（数据库配置）

- 200 积分 = 0.2 篇完整流程 或 0.5 篇 Rules+改写
- 赠送成本 ≈ ¥0.29/用户（极低，作为获客成本可忽略）

## 信息安全规则

- **CPT 值不可暴露给普通用户**（绝密）
- API 响应不返回 `total_tokens`、`estimated_tokens` 等字段
- 前端所有用量展示统一用"积分"
- 管理后台（`/api/admin/*`）可查看和修改 CPT
- 详见 CLAUDE.md "Token 与积分汇率是绝密信息" 规范
