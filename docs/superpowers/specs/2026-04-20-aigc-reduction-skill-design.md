# 论文 AIGC 降重 Skill — 设计文档

> 日期: 2026-04-20
> 状态: 待审批

## 1. 概述

### 1.1 目标

开发一个 Claude Code skill，帮助用户降低论文 AIGC 查重率。输入是用户已人工修改但仍被检测出高 AIGC 率的论文，输出是经深度改写、AIGC 率降至安全范围的论文。

### 1.2 支持平台

全平台通用，重点覆盖：
- **知网 (CNKI) 4.0**: 语义指纹 + 写作特征值，三层识别（文本-语义-逻辑），最严格
- **Turnitin**: 困惑度 + 突发性分析，支持 AI 绕过检测
- **万方**: 国产大模型全覆盖，医学/农学/工学特色检测
- **GoCheck (格子达)**: 指纹 + 语义比对，相对宽松
- **维普**: 多模型识别，速度快

**策略**: 能通过知网 4.0 的检测，其他平台大概率也能过。

### 1.3 风险阈值（四档）

| 级别 | 阈值 | 处理建议 |
|------|------|---------|
| 低风险 | <10% | 保持不动 |
| 中风险 | 10-30% | 建议微调 |
| 中高 | 30-60% | 需要改写 |
| 高风险 | >60% | 必须重写 |

## 3. AI 特征检测规则

skill 模拟检测平台的判定逻辑，扫描以下 5 类 AI 特征：

| 特征 | 检测内容 | 改写策略 |
|------|---------|---------|
| **困惑度过低** | 文本过于流畅，用词可预测性高 | 引入学科特有术语，打破词汇分布均匀性 |
| **突发性缺失** | 句式长度高度统一，缺乏波动 | 长短句交替，插入短句强调 |
| **模板化连接词** | "首先/其次/此外/综上所述" 等高频 | 用隐含逻辑替代显式连接词 |
| **认知特征缺失** | 缺乏批判性思考、个人见解、实证数据 | 注入批判性评价、第一人称分析、具体数据 |
| **语义指纹** | 语义组织方式符合 AI 逻辑 | 打乱标准论证结构，加入非标准推理路径 |

## 4. 改写风格

用户选择一种风格应用于全文改写：

| 风格 | 特点 | 适用场景 |
|------|------|---------|
| **口语化** | 日常表达，自然停顿，降低学术腔调 | 人文社科类 |
| **文言文化** | 四字成语、对仗句式 | 文科/历史/哲学 |
| **中英混杂化** | 插入英文术语、短语 | 理工科/计算机 |
| **学术人文化**（默认推荐） | 主观评价、疑问句、个人见解 | 通用 |
| **粗犷草稿风** | 短句为主，轻微语法不连贯 | 所有学科 |

### 风格示例

```
原文:「该方法能够有效提升识别准确率」

口语化:   「这个办法确实能提高识别的准确度，用起来感觉还不错」
文言文化:  「此法诚可提识别之准度，屡试不爽，成效昭然」
中英混杂:  「这个方法确实能 improve 识别的 accuracy，尤其在 edge case 下表现更好」
学术人文:  「笔者认为，该方法确实能提升识别准确率——但值得我们注意的是，在特定场景下是否依然有效？」
粗犷草稿:  「方法有用。识别率上去了。但有些情况还得再看看。」
```

## 5. 交互流程详解

### 5.1 首次扫描报告格式

```
═══ AIGC 风险评估报告 ═══
总字数: 8,500 | 预估 AIGC 率: 45%

段落 1  "随着人工智能技术的快速发展..." → "......推动了各行业的智能化转型。"
  风险: 🟢 8%  | 评价: 人类写作特征明显，句式自然波动

段落 2  "本研究旨在探讨深度学习在..." → "......为后续研究提供参考依据。"
  风险: 🔴 68% | 评价: 句式高度统一，模板化连接词过多，缺乏批判性思考

段落 3  "实验结果表明，该模型在..." → "......有效提升了识别准确率。"
  风险: 🟠 42% | 评价: 语义组织方式符合AI逻辑，论证结构过于标准

需处理段落: 段落2, 3, 7, 11, 14 (共5段)
═══════════════════════════
```

### 5.2 逐段精调交互格式

```
📍 处理进度: 1/5
段落 2: "本研究旨在探讨深度学习在..." → "......为后续研究提供参考依据。"
风险等级: 🔴 高风险 (68%)

AI 特征分析：
  • 困惑度过低：用词过于标准化（"首先/其次/此外"）
  • 突发性缺失：6 个句子长度均为 20-25 字
  • 认知特征缺失：无批判性观点，纯陈述

改写方案 A（推荐）：
  [改写后的完整段落...]

改写方案 B（保守）：
  [改写后的完整段落...]

请选择：A / B / 手动输入 / 跳过
```

```

```

## 6. 全量语义重构

### 6.1 为什么需要全量处理

知网 4.0 的语义指纹是跨段落的。局部修改会破坏上下文连贯性，容易引发系统二次预警。因此需要：

1. **打散原句骨架并重建逻辑**
2. **保持学术严谨性**，避免改完后变成"大白话"
3. **统一处理全文**，而非仅针对标红区域

### 6.2 重构策略

- **打破结构指纹**: 彻底重组段落顺序，替换程式化连接词
- **注入人类认知视角**: 代入具体学科学生身份，补充个人思辨、研究疑问
- **专业术语精准映射**: 用领域特定词汇替换通用表述
- **文献有机融合**: 观点对比、论据支撑，而非机械堆砌

## 7. 技术约束

### 7.1 检测能力

skill 不接入外部检测 API，通过内置规则引擎模拟检测平台的判定逻辑。这意味着：

- 无法 100% 保证通过任何平台的检测
- 目标是把预估 AIGC 率降至 10% 以下
- 建议用户最终用学校官方系统或免费渠道终验

### 7.2 安全阈值

因检测系统存在波动，建议将目标分数设定在校方标准以下 5%-10%（如学校要求 <20%，则需控制在 <15%）。

### 7.3 格式保护

重构过程中必须保留：
- 公式、图表结构
- 核心术语
- 学术格式规范
- 引用和参考文献格式

## 8. 输入输出

### 8.1 输入

- 论文全文（Markdown 或纯文本格式）
- 用户选择的改写风格

### 8.2 输出

- `aigc_reduced_paper.md` — 修改后全文
- `diff_report.md` — 差异对比报告
- 终端交互过程中的风险评估报告
